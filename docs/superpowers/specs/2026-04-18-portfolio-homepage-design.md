# 个人作品集首页设计文档

**日期：** 2026-04-18  
**状态：** 待开发  
**技术：** 纯 HTML / CSS / JS  
**部署：** GitHub Pages（公开链接，任何人可访问）

---

## 一、目标

为面试场景制作一个个人作品集网站首页。用户将链接发给面试官，对方直接在浏览器打开即可，无需注册或安装任何工具。

---

## 二、页面结构

首页（`index.html`）按顺序分为三个区块：

```
┌─────────────────────────────┐
│  导航栏（固定在顶部）         │
├─────────────────────────────┤
│  1. 封面 Hero               │
├─────────────────────────────┤
│  2. 简历                    │
├─────────────────────────────┤
│  3. 项目缩略                 │
└─────────────────────────────┘
```

项目详情页（`project-1.html` 等）暂缓，待首页完成后单独规划。

---

## 三、各区块设计规范

### 导航栏

- 固定在页面顶部，`position: fixed`
- 左侧：姓名 Logo（字重 700，字号 15px）
- 中间：胶囊导航，**仅含「项目」「简历」两个链接**，点击平滑滚动到对应区块
- 右侧：「联系我」按钮（黑色实心，圆角 10px）
- 背景：毛玻璃效果（`backdrop-filter: blur(12px)`，`background: rgba(250,250,250,0.8)`）
- 滚动超过 10px 后显示底部细边框

### 封面 Hero

- 全屏高度（`min-height: 100dvh`），垂直居中
- 背景：浅灰 `#f9fafb` + 网格线（64px 间距）+ 绿色光晕球（右上）+ 灰色光晕球（左下）
- 左侧内容：
  - 绿色小徽章（「正在接受新机会」，带呼吸动效绿点）
  - 超大标题（字号 clamp(3rem, 5vw, 4.5rem)，字重 800，行间距 1.05）
  - 副标题描述（17px，灰色）
  - 两个按钮：「查看项目」（黑色实心）+「查看简历」（白色描边）
  - 底部数字统计区（3 列：年经验、项目数、作品数）
- 右侧：个人信息卡片（白色圆角卡，含头像、姓名、职位、技能标签）

### 简历区块

- 背景：白色，顶部细边框与封面分隔
- 布局：左右两栏（`1fr 2fr`）
  - 左栏：个人简介文字 + 技能标签组（设计工具 / 专业能力 / 联系方式）
  - 右栏：工作 & 教育时间轴
- 时间轴：绿色圆点节点，连接线，每条记录含：时间段、职位、公司名、描述
- 右上角：「下载简历」按钮

### 项目缩略区块

- 背景：浅灰 `#fafafa`
- 2 列网格，4 张卡片（`grid-template-columns: repeat(2, 1fr)`）
- 每张卡片：
  - 顶部缩略图区域（16:9，渐变色占位，后续替换为真实截图）
  - 项目类型标签 + 年份
  - 项目名称（字重 700）
  - 一句话描述
  - 底部：技能标签 + 箭头按钮
  - 悬停效果：上浮 3px + 加深阴影 + 箭头变黑色实心
- 点击卡片：暂时无跳转（待项目详情页规划后补充）

---

## 五、技术规范

- 单文件：`index.html`（内联 CSS + JS，无外部依赖除 Google Fonts）
- 不使用任何 JS 框架或 CSS 框架
- 所有动效使用 CSS transition / animation，不依赖 JS 动画库
- 图片资源放在 `assets/images/` 目录
- 响应式：优先桌面端，移动端基础适配（导航折叠、单列布局）

---

## 六、部署方式

1. 在 GitHub 创建仓库（`portfolio`）
2. 将 `作品集/` 文件夹内容推送至 `main` 分支
3. 在 GitHub 仓库设置中开启 GitHub Pages（Source: main branch）
4. 访问地址：`https://<用户名>.github.io/portfolio`
5. 任何人可直接访问，无需登录

---

## 七、内容占位说明

当前 `index.html` 中的所有文字（姓名、经历、项目描述）均为占位内容，首页开发完成后由用户替换为真实信息。
